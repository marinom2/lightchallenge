# API Reference

All API routes are served from the Next.js webapp under `/api/`.

**Base URL:**
- UAT: `https://uat.lightchallenge.app/api`
- Local: `http://localhost:3000/api`

## Challenges

### GET /api/challenges

List challenges with optional filters.

**Query parameters:**
| Parameter | Type | Description |
|-----------|------|-------------|
| `status` | string | Filter by status (Active, Finalized, Canceled) |
| `limit` | number | Max results (default: 50) |
| `offset` | number | Pagination offset |

### POST /api/challenges

Create a new challenge (requires wallet signature headers).

### GET /api/challenge/[id]

Get full challenge details including on-chain data.

### GET /api/challenges/meta/[id]

Get challenge metadata (title, description) from the database. Fast (~100ms).

## Evidence

### POST /api/aivm/intake

Submit evidence for verification. Accepts `multipart/form-data`.

**Form fields:**
| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `json` | string | Yes | JSON evidence payload |
| `challengeId` | string | Yes | Challenge ID |
| `subject` | string | Yes | Wallet address |
| `provider` | string | Yes | Evidence source (apple-health, opendota, etc.) |
| `modelHash` | string | Yes | Model hash for verification |
| `evidenceToken` | string | No | Signed auth token (from deep link) |
| `evidenceExpires` | string | No | Token expiry timestamp |

**Authentication (one of):**
1. Wallet signature headers (`x-wallet-address`, `x-wallet-signature`, `x-wallet-message`)
2. Evidence token form fields (`evidenceToken`, `evidenceExpires`)

## Participants

### POST /api/challenge/[id]/participant

Join a challenge as a participant.

### GET /api/challenges/[id]/progress

Get participant progress for a challenge.

## Claims

### GET /api/me/claims

Get claim history for the connected wallet.

### POST /api/challenges/[id]/claim

Initiate a claim for a finalized challenge.

## User

### GET /api/me/challenges

Get the connected user's challenges.

### GET /api/me/achievements

Get the connected user's NFT achievements.

### GET /api/me/reputation

Get the connected user's reputation score.

## Platform

### GET /api/stats

Protocol-wide statistics.

**Response:**
```json
{
  "ok": true,
  "totalChallenges": 49,
  "validatorStake": "1.5",
  "modelsCount": 10
}
```

### GET /api/health

Network and service health check.

**Response:**
```json
{
  "status": "healthy",
  "rpc": true,
  "db": true,
  "blockNumber": "0x1a2b3c",
  "blockAge": 12,
  "timestamp": 1710000000
}
```

### GET /api/status?ids=1,2,3

Get on-chain status for specific challenge IDs.

## Admin

### GET/POST /api/admin/models

Manage AIVM model registry. Requires `x-admin-key` header.

### GET/POST /api/admin/templates

Manage challenge templates. Requires `x-admin-key` header.

## Authentication

### GET /api/linked-accounts

List linked gaming/fitness accounts for the connected wallet.

### GET /api/auth/steam

Initiate Steam OpenID authentication flow.

### GET /api/auth/strava

Initiate Strava OAuth flow.

### GET /api/auth/fitbit

Initiate Fitbit OAuth flow.

## RPC Proxy

### POST /api/rpc

Proxies JSON-RPC calls to the Lightchain node. Used by the webapp to avoid CORS issues.

### POST /api/relay

EIP-2771 meta-transaction relay (gasless transactions via TrustedForwarder).
