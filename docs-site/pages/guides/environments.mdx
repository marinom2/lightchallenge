# Environments

LightChallenge supports three environments: **Local**, **UAT**, and **Production**.

## Domain Architecture

| Environment | App | Docs | Explorer |
|-------------|-----|------|----------|
| **Local** | `localhost:3000` | — | — |
| **UAT** | `uat.lightchallenge.app` | `uat.docs.lightchallenge.app` | [testnet.lightscan.app](https://testnet.lightscan.app) |
| **Production** | `app.lightchallenge.app` | `docs.lightchallenge.app` | TBD |

## Chain Configuration

| Environment | Chain | Chain ID | RPC |
|-------------|-------|----------|-----|
| Local | Lightchain Testnet | 504 | `https://light-testnet-rpc.lightchain.ai` |
| UAT | Lightchain Testnet | 504 | `https://light-testnet-rpc.lightchain.ai` |
| Production | Lightchain Mainnet | TBD | TBD |

## Database

Each environment uses a separate PostgreSQL database (Neon):

| Environment | Variable | Notes |
|-------------|----------|-------|
| Local | `DATABASE_URL` in `webapp/.env.local` | Developer's local or Neon branch |
| UAT | `DATABASE_URL` in Vercel (Preview) | Shared UAT database |
| Production | `DATABASE_URL` in Vercel (Production) | Production database (separate Neon project) |

## Key Environment Variables That Differ

| Variable | Local | UAT | Production |
|----------|-------|-----|------------|
| `NEXT_PUBLIC_BASE_URL` | `http://localhost:3000` | `https://uat.lightchallenge.app` | `https://app.lightchallenge.app` |
| `DATABASE_URL` | Local/dev DB | UAT Neon DB | Production Neon DB |
| `LCAI_WORKER_PK` | Dev wallet | UAT operator wallet | Production operator wallet |
| `LCAI_FINALIZE_PK` | Dev wallet | UAT operator wallet | Production operator wallet |
| `NEXT_PUBLIC_CHAIN_ID` | `504` | `504` | TBD (mainnet) |

## Vercel Configuration

Single Vercel project (`lightchallenge`) with environment-scoped variables:

- **Preview deployments** → UAT environment variables
- **Production deployments** → Production environment variables
- Root directory: `webapp`
- Framework: Next.js
- GitHub auto-deploy: `main` branch

## iOS App

| Environment | Server URL | Default |
|-------------|-----------|---------|
| Local | `http://<MAC_IP>:3000` | No |
| UAT | `https://uat.lightchallenge.app` | Yes (testnet phase) |
| Production | `https://app.lightchallenge.app` | Yes (mainnet phase) |

The iOS app defaults to UAT during the testnet phase. Users can switch environments in **Profile → Settings**.

### iOS Domain References

The following must be updated when switching from UAT to production:

| File | What to change |
|------|----------------|
| `Sources/Models/Models.swift` | `ServerConfig.defaultBaseURL` |
| `Sources/LightChallengeApp.swift` | Universal link comment |
| `Sources/LightChallengeApp.entitlements` | Associated domains (`applinks:`) |
| `Sources/Views/Explore/ExploreView.swift` | Desktop handoff text |

### TestFlight Distribution

The iOS app is ready for TestFlight beta distribution:

1. **Archive**: In Xcode, select **Product → Archive** (requires a physical device or "Any iOS Device" target)
2. **Upload**: In the Organizer window, click **Distribute App → App Store Connect → Upload**
3. **TestFlight portal**: Go to [App Store Connect](https://appstoreconnect.apple.com) → My Apps → LightChallenge → TestFlight
4. **Add testers**: Create an internal or external testing group, add tester emails
5. **External testing**: Requires a brief App Review (~24-48 hours for first build)

**Notes:**
- Bundle ID: `io.lightchallenge.app`
- Requires Apple Developer Program membership (already active)
- Each build must increment `CFBundleVersion` (currently `2` in Info.plist)
- HealthKit usage description in Info.plist is required for review approval

## Local Development Setup

```bash
# 1. Clone and install
git clone https://github.com/marinom2/lightchallenge.git
cd lightchallenge
npm install
cd webapp && npm install && cd ..

# 2. Configure environment
cp .env.example webapp/.env.local
# Edit webapp/.env.local with your DATABASE_URL and keys

# 3. Run database migrations
npx tsx db/migrate.ts

# 4. Start the webapp
cd webapp && npm run dev

# 5. Start workers (separate terminals)
npx tsx offchain/workers/evidenceEvaluator.ts
npx tsx offchain/workers/challengeDispatcher.ts
npx tsx offchain/workers/challengeWorker.ts
npx tsx offchain/indexers/aivmIndexer.ts
```

## UAT → Production Migration Checklist

When testing is complete and the project is ready for production:

### 1. Domain Switch

| Component | UAT | Production |
|-----------|-----|------------|
| Webapp | `uat.lightchallenge.app` | `lightchallenge.app` or `app.lightchallenge.app` |
| Docs | `uat.docs.lightchallenge.app` | `docs.lightchallenge.app` |
| iOS default URL | `https://uat.lightchallenge.app` | `https://app.lightchallenge.app` |
| Universal links | `applinks:uat.lightchallenge.app` | `applinks:lightchallenge.app` |
| AASA file | Served from UAT domain | Must be served from production domain |

### 2. Vercel

- Update production domain in Vercel project settings
- Set production environment variables (`DATABASE_URL`, `LCAI_WORKER_PK`, `LCAI_FINALIZE_PK`, `NEXT_PUBLIC_BASE_URL`)
- Verify `webapp/public/.well-known/apple-app-site-association` is served correctly on the production domain

### 3. Smart Contracts

- Deploy fresh contracts to mainnet (new chain ID, new RPC)
- Update `webapp/lib/contracts.ts` and `deployments/` for mainnet addresses
- Update iOS `Sources/Models/Contracts.swift` with mainnet addresses and chain config

### 4. iOS App

- Update `ServerConfig.defaultBaseURL` in `Models.swift` → production URL
- Update associated domains in entitlements → `applinks:lightchallenge.app`
- Update `Contracts.swift` → mainnet chain ID, RPC, contract addresses
- Increment `CFBundleVersion` and `CFBundleShortVersionString`
- Archive and upload new build to App Store Connect
- Submit for App Store review (if going public) or distribute via TestFlight

### 5. Database

- Create production Neon database (separate project from UAT)
- Run all migrations: `DATABASE_URL=<prod_url> npx tsx db/migrate.ts`
- Do NOT copy UAT data — production starts fresh

### 6. Workers

- Deploy workers pointing to production DB and mainnet RPC
- Update `ecosystem.config.cjs` if using PM2
- Verify all indexers are tracking mainnet contract events

### 7. DNS

- Point production domains to Vercel
- Ensure HTTPS certificates are provisioned
- Verify AASA file is accessible at `https://lightchallenge.app/.well-known/apple-app-site-association`
